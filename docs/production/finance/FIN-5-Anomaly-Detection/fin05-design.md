# Design Specifications: Anomaly Detection (FIN-5)

## 1. Screen Overview
**Page:** Anomaly Detection Dashboard

## 2. Wireframe Description
```text
=== Financial Watchdog / Anomalies ===
[Header] Last Scan: Today at 02:00 AM | Active Alerts: 2

[Data Table]
| Date | Entry No | Account | Amount | AI Assessment | Actions |
| 15/05| JE-99    | 5100    | 50M    | [Sparkle Icon] Amount is 25x historical average... | [Resolve] [View Entry] |
| 16/05| JE-102   | 1110    | 1M     | [Sparkle Icon] Posted at 03:00 AM on a Sunday...   | [Resolve] [View Entry] |

=== Resolve Modal ===
Title: Resolve Anomaly (JE-99)
[AI Assessment text repeated]

Mark as:
( ) Valid Transaction (Ignore)
( ) Suspicious (Requires Investigation)
( ) Erroneous (Requires Reversal)

[Notes Textarea]
[Button: Submit]
```

## 3. Component Hierarchy
- `AnomalyDashboard`
  - `StatusHeader`
  - `AnomalyTable`
    - `AiSummaryTooltip` (Hover to view full AI text)
    - `ActionButtons`
  - `ResolveAnomalyDialog`
    - `ResolutionForm` (Radio buttons + text area)

## 4. UI Components (shadcn/ui)
- `Table` for the alerts list.
- `Badge` with a distinct color (e.g., Purple or Gold) and an icon (sparkles) to denote AI-generated content.
- `Dialog` for the resolution workflow.
- `RadioGroup` for resolution categories.

## 5. Visual Guidelines
- **AI Branding**: Use consistent visual cues (purple color, sparkle icons) across the ERP whenever content is generated by the Groq AI, distinguishing it from deterministic system data.
- **Urgency**: Active anomalies should feel like an inbox that needs to be cleared (Inbox Zero concept).

## 6. Responsive Design
- Desktop-first layout, as auditing is typically a desktop activity.

## 7. States & Interactions
- **Resolve Interaction**: Resolving an anomaly removes the row from the default view with a smooth slide-up animation.
- **Loading State**: Shimmer loading effects if the anomalies are being fetched from the server.

## 8. Accessibility
- Ensure the AI tooltips are accessible via keyboard focus (`Tab`).
- Use clear ARIA labels for the resolution radio buttons.
